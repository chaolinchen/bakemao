# BakeMao 烘焙貓 PRD
**版本：v1.4 | 日期：2026-04-17 | 狀態：確認**

---

## 1. 問題定義

**現況痛點：** 烘焙時需要手算「容器容量 → 總克數 → 各材料克數」，涉及多步公式（烘焙百分比、耗損係數、模具換算），容易出錯、費時。大多數人手邊的食譜是克數格式，不是百分比，換模具或改份量時需要重新手算。

**我們的解法：**
- **模式 A（百分比 → 克數）：** 輸入配方百分比 + 模具/目標克數，一鍵算出各材料克數
- **模式 B（克數 → 百分比 → 新克數）：** 輸入現有克數配方，系統反推百分比，換模具或改目標克數後自動重算

搭配常見模具快選、食材 Autocomplete、品牌資料庫，大幅降低輸入門檻。

## 2. 目標用戶

**主要用戶：** 業餘到認真的烘焙愛好者
- 手邊有食譜（克數或百分比皆可）
- 習慣用手機在廚房查食譜
- 想換模具尺寸、改份量時不需要手算

**次要用戶：** 烘焙檢定考生（需依考題版本快速計算，熟悉百分比）

- 計算完成率 > 80%（開始計算到看到結果）

## 4. 核心計算邏輯

### 模式 A：百分比 → 克數
```
目標容量(g) ÷ 耗損係數 ÷ 配方總百分比 = 每 1% 對應的克數
每 1% × 各材料百分比 = 各材料的克數
```

目標容量來源（二選一）：
- **選模具**：模具容積(cc) × 數量（cc ≈ g）
- **直接輸入目標總克數**（適合戚風蛋糕等不看容器的配方）

### 模式 B：克數 → 百分比（反推）
```
各材料克數 / 總克數 × 100 = 各材料百分比
```
反推完成後進入與模式 A 相同的計算引擎，用戶只需更換目標克數或模具即可重算。

### 特殊材料類型
- **比例材料**：隨配方縮放（如：鮮奶、蛋液）
- **固定材料**：不隨數量縮放，不計入百分比（如：焦糖砂糖固定 100g）

## 5. MVP 功能範圍

### ✅ 納入 MVP（P0）

| 功能 | 說明 |
|------|------|
| **模式 A：百分比輸入** | 各材料輸入名稱 + 百分比，即時計算克數 |
| **模式 B：克數反推** | 輸入現有配方克數，自動反推百分比後可換算 |
| **目標設定（模具）** | 圓柱形、長方形、馬芬、直接輸入 cc |
| **目標設定（直接克數）** | 直接輸入目標總克數，適合戚風等配方 |
| **常見模具快選** | 內建常見規格，點選自動帶入 |
| **食材 Autocomplete** | 關鍵字搜尋，內建 50～80 種常見烘焙食材 |
| **食材品牌快選（內建）** | 選定食材後可選常見品牌，帶入已知理化數值 |
| **固定 vs 比例材料** | 材料可切換「固定克數」或「依百分比縮放」|
| **耗損係數設定** | 新手顯示「備用 +0/+1/+2」，背後換算係數；進階模式可直接輸入 %  |
| **計算結果顯示** | 各材料克數（字體 ≥28px）+ 橫條比例視覺化 |

### ✅ 納入 MVP（P1）

| 功能 | 說明 |
|------|------|
| **PWA 離線支援** | 核心計算邏輯離線可用（Service Worker） |
| **Wake Lock 螢幕常亮** | 進入計算頁後螢幕不熄滅，離開後釋放 |

| **配方管理** | 查看、編輯、刪除已儲存配方 |
| **分享結果圖** | 計算完成後生成含材料克數的圖片，一鍵存圖/發 IG Story |

## 8. 非功能需求

- **效能：** 計算結果即時顯示（輸入即算，無 loading）；離線核心功能可用
- **UX：** 按鈕最小 48px；主要操作在螢幕下半部（拇指可達區）；數字輸入自動喚起數字鍵盤；高對比模式
- **廚房場景：** 單手操作優先；食材搜尋用 Bottom Sheet；克數字體最小 28px Bold
- **相容性：** iOS Safari 16+、Android Chrome 108+
- **部署：** bakemao.smallfatmao.com，Vercel + Next.js
- **資料庫：** Supabase（與生態系一致，非 Firestore）

## 14. 計算引擎狀態機

```
初始狀態
  ├─ 選擇「我有百分比配方」→ 模式 A
  └─ 選擇「我有克數配方」  → 模式 B

模式 A：百分比輸入 → 設定目標 → 即時計算 → 顯示結果
模式 B：輸入克數 → 自動反推% → 設定新目標 → 即時計算 → 顯示結果

切換模式：需 confirm dialog 清空現有輸入
儲存的配方記錄模式類型，載入時還原對應模式
```

## 16. 邊界條件與 Error State

| 情況 | 行為 |
|------|------|
| 輸入 0 | 紅框 + 提示「請輸入大於 0 的數字」，不阻擋其他計算 |
| 輸入負數 | 自動轉絕對值，不報錯 |
| 材料清單為空 | 空狀態插圖，不顯示計算結果 |
| 目標克數為 0 | 所有比例材料顯示 0g，固定材料不變 |
| 模具尺寸不完整 | 容積顯示「—」，結果不更新 |
| 離線時按儲存 | Toast「目前離線，連線後自動儲存」，暫存 localStorage |

## 17. 離線行為規格

| 功能 | 離線可用 | 說明 |

## 18. 登入 / 儲存流程

```
點擊「儲存配方」
  ├─ 已登入 → 顯示命名 Dialog → 儲存 → Toast「已儲存 ✓」
  └─ 未登入 → Bottom Sheet（Google / Apple 登入）
                  → 登入成功 → 自動執行儲存

配方命名：預設「配方 YYYY/MM/DD HH:mm」，最多 30 字

Supabase recipes 資料表欄位：
  id, user_id, name, mode, target_type, target_gram,
  mold_id, mold_params(JSON), quantity,
  loss_type, loss_value, ingredients(JSON),
  created_at, updated_at
  + RLS：用戶只能存取自己的配方
```
